# 🚨 긴급 수정 사항 (v2.2)

## 발견된 심각한 버그

### 1. 수수료율 입력 버그 ⚠️
**증상:**
```
사용자 입력: 0.05
기대 결과: 0.05%
실제 결과: 5.00% (100배!)
```

**영향:**
- 모든 거래에서 비현실적인 수수료 부과
- 수익률 계산이 완전히 왜곡됨

**수정:**
```python
# Before
fee_rate = float(input)  # 0.05 입력 → 0.05 (5%)

# After  
fee_rate = float(input) / 100  # 0.05 입력 → 0.0005 (0.05%)
```

## 즉시 테스트 방법

### 테스트 1: 수수료율 확인
```
종목: 삼성전자
기간: 1년
전략: 1 (SMA)
수수료: 0.05 입력
쿨다운: 0
주문비율: 0.3
```

**확인 포인트:**
- "수수료 0.050%" 표시 확인 (5%가 아님!)

### 테스트 2: 거래 발생 확인
```
종목: 삼성전자
기간: 1년
전략: 1 (SMA)
모두 Enter (기본값)
```

**확인 포인트:**
- 거래가 발생해야 함
- 안 되면 신호 분석 확인:
  ```
  • BUY 신호 발생: ?회
  • 쿨다운에 막힘: ?회
  ```

### 테스트 3: 모멘텀 전략 (가장 활발)
```
종목: 엔비디아 (10번)
기간: 6개월 (3번)
전략: 6 (모멘텀)
모두 Enter
```

**예상 결과:**
- 거래 3회 이상 발생
- 수익률 표시

## 주요 개선 사항

### 1. 전략 설명
이제 각 전략마다:
- 📌 간단 설명
- ✅ 언제 매수하는지
- ❌ 언제 매도하는지
- 💡 어떤 종목에 적합한지

### 2. 거래 없을 때
정확한 원인 파악:
```
📊 신호 발생 분석:
   • BUY 신호 발생: 15회
   • SELL 신호 발생: 8회
   • 쿨다운에 막힘: 12회  ← 주요 원인!
```

### 3. 기본값 변경
- 쿨다운: 1일 → 0일 (더 활발한 거래)
- 주문비율: 20% → 30% (더 공격적)

## 권장 사용법

### 초보자
```
1. 전략 메뉴 꼭 읽기 (각 전략 설명 확인)
2. 기본값 그대로 사용 (Enter 연타)
3. 결과 확인
4. 신호 분석 보고 개선
```

### 고급 사용자
```
1. 쿨다운 0으로 설정
2. 주문비율 0.5 (50%)
3. 여러 전략 비교
4. 랭킹에서 최고 전략 확인
```

## 문제 해결 가이드

### Q: 여전히 거래가 안 돼요
```
A: 신호 분석 확인
   - BUY/SELL 신호가 0회면 → 다른 전략 시도
   - 쿨다운에 막힘이 많으면 → 쿨다운 0 설정
   - 현금 부족이면 → 주문비율 낮추기
```

### Q: 수수료가 이상해요
```
A: % 단위로 입력하세요
   - 0.05 입력 = 0.05%
   - 0.015 입력 = 0.015% (한국 증권사 평균)
   - 0 입력 = 0% (미국 주식)
```

### Q: 전략을 어떻게 선택하나요?
```
A: 전략 메뉴의 설명 참고
   1. SMA: 초보자, 추세 명확한 종목
   3. RSI: 횡보장, 변동성 큰 종목
   6. 모멘텀: 강한 추세 종목
```
